File permissions in Linux

Project description

Here is a directory belonging to a research team of a large organization. To ensure users of the team are authorized with the appropriate permissions to keep the system secure, the principle of least privilege is implemented through the use of Linux commands.

The files and directories belonging to the research team are analyzed. Permissions are added where it is needed and permissions are removed where it is not necessary to abide by the regulations of the organization.

Check file and directory details

The commands used to check the files and the directories are as follows:

The command used here to check file and directory permissions is: ls -la

This command shows the files/directories inside the /home/researcher2/projects directory and includes hidden files and shows detailed information about the permissions, owner, owner group, size of the file/directory, date and time of last modification and the file/directory names in the output screen.

Describe the permissions string

From the output above we can see that there is a directory called drafts. The permissions for this directory are represented by the string:  drwx--x---.

Here, the first character of the string is d. It represents that it is a directory.

The next three characters of the string represent the permissions for the owner/user of the directory:

rwx : r means the user has read permission, w means the user has write permission, and x means that the directory is modifiable by the user

The next three characters of the string represent the permissions for the group the directory belongs to:

--x : - means the group does not have read permission, the next - means the group does not have write permission, and the next character x means that the group has access to the directory and view and search inside the directory

The last three characters of the string represent permissions for the other users of the directory:

--- : - means other users do not have read permission, the next - means other users do not have write permission, and the next character - means that other users do not have access to the directory. They cannot view or search inside the directory.

Change file permissions

From the section Check file and directory details above, the file project_k.txt has write permission set to other users. As per the organization rules, other users shall not have write permission to any of the files. Hence the command below is used to remove the write permission:

chmod o-w project_k.txt

This command removes the write permission from user type other for the file project_k.txt.

Change file permissions on a hidden file

The file .project_x.txt is archived by the research team, hence the file is hidden. No user type shall have write permission for this file. Although only the user and group shall be able to read.

From the section Check file and directory details above, the file .project_x.txt has write permission enabled for the user and the group but does not have read permission enabled for the group users. To rectify this non-compliance, below command is used:

chmod u-w,g-w,g+r .project_x.txt

This command removes the write permissions from the user and the group and adds read permission for the group.

Change directory permissions

The drafts directory mentioned in the section Describe the permissions string should only belong to the user researcher2. Since the group has access to the drafts directory as stated above, the access permission needs to be removed. It is done by executing the following command:

chmod g-x drafts

This command removes the access permission for the group and settles the non-compliance issue.

Summary

The tasks performed in this instance ensure the principle of least privilege applied throughout the directory /home/researcher2. The addition and removal of read, write, and access/execute permissions ensure the safety of the content inside the directory. It also ensures that identity and access management is done properly through the use of Linux commands in the bash shell.

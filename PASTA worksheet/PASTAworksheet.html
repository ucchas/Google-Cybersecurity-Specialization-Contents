I. Define business and security objectives

Make 2-3 notes of specific business requirements that will be analyzed.

  • Will the app process transactions?
  • Does it do a lot of back-end processing?
  • Are there industry regulations that need to be considered?

The app will process transactions that require them to follow industry regulations such as PCI DSS. Also there is a lot of information handling to be conducted on the back-end like Q&A, seller reviews, customer data, and payment processing.

II. Define the technical scope

List of technologies used by the application:

  • Application programming interface (API)
  • Public key infrastructure (PKI)
  • SHA-256
  • SQL

Write 2-3 sentences (40-60 words) that describe why you choose to prioritize that technology over the others.

As SQL is used to handle the database, it needs to be evaluated first for SQL injection attacks. Various types of user input can make the database vulnerable to outside threat actors. API technologies used for the application also need to be evaluated for proper use and other types of vulnerabilities.

IV. Threat analysis

List 2 types of threats in the PASTA worksheet that are risks to the information being handled by the application.

  • What are the internal threats?
  • What are the external threats?

Disgruntled employees, sellers, and even any on-premise worker can be an internal threat to the company. Whereas, other business competitors, malicious actors/hackers, and government agencies can pose an external threat.

V. Vulnerability analysis

List 2 vulnerabilities in the PASTA worksheet that could be exploited.

  • Could there be things wrong with the codebase?
  • Could there be weaknesses in the database?
  • Could there be flaws in the network?

The way the codebase is structured can be exploited. Codebase formatting needs to be checked for SQL injection attack methods. The use of SHA-256 along with the AES and RSA algorithms where needed is to be verified. The APIs used also shall be looked upon for possible attack vectors.
